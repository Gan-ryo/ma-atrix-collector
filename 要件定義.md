# MA-ATRIX News Collector 要件定義

> **⚠ 注意：仕様を変更した場合は必ずこのファイルも更新すること。**

最終更新: 2026-05-24

---

## 1. プロジェクト概要

### 目的
PR TIMES / BusinessWire の RSS フィードから AI 関連記事を自動収集し、
各記事が MA-ATRIX（生成AI活用組織成熟度フレームワーク）のどの評価軸・成熟度レベルに
相当するかを Gemini API で自動分類して CSV に蓄積する。

### 背景・制約
- 組織の GCP 環境で `iam.disableServiceAccountKeyCreation` ポリシーが適用されており、
  サービスアカウントキーの作成が不可のため Google Sheets は利用できない。
- 追加のクラウドサービスを使わず、GitHub リポジトリへの CSV コミットで蓄積する。

---

## 2. システム構成

```
GitHub Actions（毎日 JST 10:00 / UTC 01:00 に定期実行）
    ↓
collector.py
    ├─ feedparser     → RSS フィード取得（PR TIMES / BusinessWire）
    ├─ キーワードフィルタ → AI 関連記事のみ抽出
    ├─ Gemini 1.5 Flash API → MA-ATRIX 評価軸・レベル分類
    └─ csv モジュール  → data/articles.csv に追記
    ↓
git commit & push（新規記事があった場合のみ）→ main ブランチ
```

### 使用技術・サービス

| 役割 | 技術・サービス | バージョン | 備考 |
|---|---|---|---|
| 定期実行 | GitHub Actions | - | 無償枠内 |
| RSS 取得 | feedparser | 6.0.12 | PyPI |
| AI 分類 | Gemini 1.5 Flash API | - | 無償枠: 15RPM / 1,500RPD |
| AI SDK | google-genai | 2.6.0 | PyPI（旧 google-generativeai は 2025-11-30 サポート終了） |
| 蓄積 | CSV（リポジトリコミット） | - | `data/articles.csv` |
| Python | Python | 3.11 | GitHub Actions runner |
| ランナー OS | ubuntu-latest | - | GitHub Actions |

## 4. RSS フィード仕様

### 収集対象フィード

| ソース名 | URL | フィルタ |
|---|---|---|
| PR TIMES | `https://prtimes.jp/index.rdf` | AI キーワードでフィルタ（全件 RSS のため必須） |
| PR Newswire | `https://www.prnewswire.com/rss/news-releases-list.rss` | AI キーワードでフィルタ（全件 RSS のため必須） |

> **変更履歴**：旧 PR TIMES `rss.php` は 404 のため `index.rdf` に変更。旧 BusinessWire AI フィードはエントリ 0（事実上廃止）のため PR Newswire 全件フィードに変更。

### AI キーワードフィルタ

以下のいずれかのキーワードをタイトルまたは概要に含む記事のみを処理対象とする。
キーワードマッチは大文字/小文字を区別しない（`.lower()` で統一）。

**日本語キーワード：**
`AI`、`人工知能`、`生成AI`、`ChatGPT`、`LLM`、`機械学習`、`ディープラーニング`、
`深層学習`、`大規模言語モデル`、`自動化`、`Claude`、`Gemini`、`Copilot`、
`チャットボット`、`自然言語`、`RAG`、`エージェント`

**英語キーワード：**
`artificial intelligence`、`generative AI`、`machine learning`、`deep learning`、
`large language model`、`neural network`、`NLP`、`natural language processing`、
`agentic`、`foundation model`

### 処理上限
- 1 フィードあたり最大 **50 件**（`MAX_PER_FEED = 50`）
- フィルタ後の上位 50 件を処理。超過分はスキップ。

## 5. Gemini API 分類仕様

### SDK・クライアント初期化
- SDK パッケージ: `google-genai`（旧 `google-generativeai` は 2025-11-30 サポート終了）
- インポート: `from google import genai`
- 初期化: `client = genai.Client(api_key=GEMINI_API_KEY)`

### モデル
`gemini-1.5-flash`

### API 呼び出し形式
```python
response = client.models.generate_content(
    model="gemini-1.5-flash",
    contents=prompt,
)
raw = response.text.strip()
```

### レート制限対応
- リクエスト間隔: **4.5 秒以上**（`GEMINI_INTERVAL_SEC = 4.5`）
- 無償枠上限: 15 RPM / 1,500 RPD に対応

### 入力プロンプト
記事のタイトルと概要（先頭200字）を以下の形式で渡す：

```
タイトル: {title}
概要: {summary}
```

### 出力形式（JSON）
```json
{
  "not_relevant": false,
  "axes": {
    "組織": null,
    "制度・仕組み": null,
    "コンプライアンス": null,
    "生成AI活用対象の業務プロセス": null,
    "データマネジメント": null,
    "生成AI活用": null,
    "生成AIの業務プロセスへの統合": null
  },
  "primary_axis": "評価軸名",
  "primary_level": 0,
  "reasoning": "判定根拠（80字以内）"
}
```

- `not_relevant: true` の場合、CSV には追記しない
- `axes` の各軸の値は成熟度レベル整数（0〜5）または `null`（無関係）
- JSON のみを返させる（コードブロック記号 ` ``` ` は含めない）
- レスポンス中の ` ```json ` 等は正規表現で除去してから `json.loads()` に渡す

## 6. CSV 出力仕様

### 保存先
`data/articles.csv`（リポジトリルートからの相対パス）

### 文字コード
UTF-8

### 列定義

| 列名 | 内容 | 生成元 |
|---|---|---|
| `記事ID` | URL の MD5 ハッシュ先頭 12 文字（重複防止キー） | `hashlib.md5(url).hexdigest()[:12]` |
| `収集日時(UTC)` | 実行時刻（UTC）`YYYY-MM-DD HH:MM` | `datetime.now(timezone.utc)` |
| `ソース` | `PR TIMES` または `BusinessWire` | RSS フィード定義 |
| `タイトル` | 記事タイトル（前後の空白除去済み） | RSS `title` |
| `URL` | 記事 URL | RSS `link` |
| `公開日` | RSS 掲載日時（文字列そのまま） | RSS `published` または `updated` |
| `概要(先頭200字)` | HTML タグ除去後の先頭 200 字 | RSS `summary` または `description` |
| `軸1_組織` | 評価軸「組織」の推定レベル（0〜5 または空白） | Gemini `axes.組織` |
| `軸2_制度仕組み` | 評価軸「制度・仕組み」の推定レベル | Gemini `axes.制度・仕組み` |
| `軸3_コンプライアンス` | 評価軸「コンプライアンス」の推定レベル | Gemini `axes.コンプライアンス` |
| `軸4_業務プロセス` | 評価軸「生成AI活用対象の業務プロセス」の推定レベル | Gemini `axes.生成AI活用対象の業務プロセス` |
| `軸5_データマネジメント` | 評価軸「データマネジメント」の推定レベル | Gemini `axes.データマネジメント` |
| `軸6_生成AI活用` | 評価軸「生成AI活用」の推定レベル | Gemini `axes.生成AI活用` |
| `軸7_業務統合` | 評価軸「生成AIの業務プロセスへの統合」の推定レベル | Gemini `axes.生成AIの業務プロセスへの統合` |
| `主要評価軸` | 最も関連性の高い評価軸名 | Gemini `primary_axis` |
| `推定レベル` | 主要評価軸の成熟度レベル（0〜5） | Gemini `primary_level` |
| `判定根拠` | Gemini による 80 字以内の説明 | Gemini `reasoning` |

### 重複防止
- 実行開始時に既存 CSV を読み込み、`記事ID` のセットを構築する
- 既存 ID に一致する記事はスキップ（Gemini API を呼ばない）

### ヘッダ出力
- CSV が存在しない場合のみ 1 行目にヘッダを書き込む
- 追記モード（`open("a")`）で運用

## 7. GitHub Actions ワークフロー仕様

### ファイルパス
`.github/workflows/collect.yml`

### トリガー
| トリガー | 設定 |
|---|---|
| スケジュール実行 | `cron: '0 1 * * *'`（毎日 UTC 01:00 = JST 10:00） |
| 手動実行 | `workflow_dispatch`（GitHub UI から「Run workflow」） |

### ジョブ設定
| 項目 | 値 |
|---|---|
| ランナー | `ubuntu-latest` |
| タイムアウト | 30 分 |
| 権限 | `contents: write`（CSV コミット・プッシュに必要） |

### ステップ

| ステップ名 | 処理 |
|---|---|
| Checkout | `actions/checkout@v5`（Node.js 24 対応版） |
| Set up Python 3.11 | `actions/setup-python@v6`（Node.js 24 対応版）、pip キャッシュ有効 |
| Install dependencies | `pip install -r requirements.txt` |
| Run collector | `python collector.py`（`GEMINI_API_KEY` を Secret から注入） |
| Commit and push results | `git add data/`（`articles.csv` 非存在時も対応）、差分あり時のみコミット、`git push` |

### コミットメッセージ形式
```
chore: collect articles YYYY-MM-DD
```
コミットユーザー: `github-actions[bot]`

## 9. MA-ATRIX フレームワーク定義

詳細は `ma-atrix-def.md` を参照。以下は分類処理で使用する評価軸とレベルの概要。

### 評価軸（7軸）

| No | 評価軸名 | 観点の要約 |
|---|---|---|
| 1 | 組織 | ビジョン・戦略・経営層コミットメント・組織文化・予算体制 |
| 2 | 制度・仕組み | 体制・ルール・教育プログラム・業務統合の制度的枠組み |
| 3 | コンプライアンス | 法令遵守・AI 倫理・個人情報・著作権・情報セキュリティ |
| 4 | 生成AI活用対象の業務プロセス | 対象業務プロセスの定義・標準化・品質管理 |
| 5 | データマネジメント | データ収集・品質・アクセス性・管理体制 |
| 6 | 生成AI活用 | モデル選定・運用・監視・評価（LLMOps/GenAIOps） |
| 7 | 生成AIの業務プロセスへの統合 | 業務プロセスへの AI 組み込み・全体最適化 |

### 成熟度レベル（0〜5）

| レベル | 定義 |
|---|---|
| 0 | 未実施・把握なし |
| 1 | 属人的・場当たり的な実施 |
| 2 | 基本方針・手順が明文化・管理 |
| 3 | 全社的に標準化・体系化・一貫運用 |
| 4 | 定量的に管理・統計的分析・継続改善 |
| 5 | 継続的最適化・迅速な変化対応 |

> ※ `ma-atrix-def.md` にはレベル 6（AI による自律的最適化）の定義も存在するが、
> 現在の分類プロンプトはレベル 0〜5 を対象としている。

## 10. エラーハンドリング仕様

| エラー種別 | 処理 |
|---|---|
| Gemini API エラー（例外） | `[WARN]` を標準出力に出力し、該当記事をスキップ |
| Gemini レスポンスの JSON 解析失敗 | `[WARN]` を標準出力に出力し、該当記事をスキップ |
| `not_relevant: true` | MA-ATRIX 無関連と判定し、CSV に追記しない |
| 新規記事ゼロ（差分なし） | git commit をスキップ（`git diff --cached --quiet` で判定） |
| `data/articles.csv` が未存在 | 初回実行時に自動作成。`git add data/` で対応済み |
